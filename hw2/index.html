Overview

In this homework I implemented Bezier curves and surfaces using de Casteljau subdivision, then moved into triangle mesh operations using the half-edge data structure. For the mesh side, I implemented area-weighted vertex normals for smooth Phong shading, edge flips, edge splits, and finally loop subdivision for mesh upsampling. The most interesting part was seeing how all three mesh operations build on each other: edge flip and split are used directly inside the loop subdivision algorithm, and getting the pointer reassignments exactly right in each operation was critical for everything downstream to work. I also found it satisfying to see how pre-processing a cube with symmetric edge splits before subdivision produces a rounder, more symmetric result compared to the default asymmetric behavior.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm evaluates a Bezier curve at a given parameter t by repeatedly applying linear interpolation between adjacent control points. Given n control points, one step of the algorithm produces n-1 new intermediate points, where each new point is computed as (1-t)*p_i + t*p_{i+1}. Applying this step recursively reduces the number of points by one each time until only a single point remains, which is exactly the point on the curve at parameter t.

My implementation of BezierCurve::evaluateStep takes the current vector of points and iterates through adjacent pairs, pushing the interpolated result into a new vector. Each call performs exactly one level of subdivision.

I created a custom 6-point Bezier curve for the screenshots below.

Part 2: Bezier surfaces with separable 1D de Casteljau

Bezier surfaces extend the 1D algorithm by applying de Casteljau in two directions. Given an n x n grid of control points and parameters u and v, I first apply evaluate1D across each row using parameter u, which produces n points. Those n points are then treated as the control points of a new curve, and evaluate1D is applied one more time using parameter v to get the final surface point. My implementation of evaluateStep for patches is identical in structure to Part 1 but operates on 3D vectors with an explicit t parameter. evaluate1D calls evaluateStep in a loop until one point remains. evaluate runs evaluate1D on each row with u, collects the results, then calls evaluate1D once more with v.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To compute an area-weighted vertex normal, I iterate over all faces incident to the vertex using the half-edge structure. Starting from the vertex's half-edge, I traverse outgoing half-edges by repeatedly following twin()->next(). For each non-boundary face, I grab the three vertex positions and compute the cross product of two edge vectors. The cross product gives a vector whose direction is the face normal and whose magnitude is proportional to twice the triangle's area, so summing these cross products automatically weights by area. At the end I normalize the sum.

Part 4: Edge flip

For edge flip, I started by carefully labeling every half-edge, vertex, edge, and face in the two-triangle configuration before the flip. Given triangles (a,b,c) and (c,b,d) sharing edge b-c, flipping produces triangles (a,d,c) and (d,a,b). No new elements are created; only pointers are reassigned. I used setNeighbors to update all six half-edges at once, then updated the halfedge pointers for all four vertices, both faces, and the edge itself. The key insight was to reassign every pointer even if it hadn't changed, which made debugging much easier since I could rule out stale pointers as the source of any issues. Boundary edges are skipped immediately at the top of the function.

Part 5: Edge split

Edge split is more involved than flip because new elements have to be created. For the two triangles (a,b,c) and (c,b,d), splitting edge b-c inserts a midpoint vertex m and produces four triangles: (a,b,m), (b,d,m), (m,c,a), and (m,d,c). I created one new vertex, three new edges, two new faces, and six new half-edges, then carefully set all neighbor pointers for every element in the new configuration. I also set isNew=true on the two cross-edges (m-a and m-d) and isNew=false on the two halves of the original split edge so that the upsample function in Part 6 knows which edges to flip. The new vertex m's halfedge is set to point in the direction of the original split edge per the spec.

The main debugging challenge was keeping track of which half-edge pointed where after the split. Drawing out the full diagram with all 10 half-edges labeled before writing any code made it much easier to get right on the first try.

Part 6: Loop subdivision for mesh upsampling

My loop subdivision implementation follows four steps. First, I compute new positions for all existing vertices using the Loop subdivision weighting formula (1 - n*u) * original_position + u * neighbor_sum, where u is 3/16 for degree-3 vertices and 3/(8n) otherwise. Second, I compute the position each new edge midpoint vertex will get using 3/8*(A+B) + 1/8*(C+D). Both of these are computed before any topology changes so that all weights are based on the original mesh. Third, I split every original edge in the mesh (by first collecting them into a separate vector to avoid iterating over newly created edges) and assign each new midpoint vertex its precomputed position. Fourth, I flip any new edge that connects one old and one new vertex. Finally, I copy all newPosition values into position.

After loop subdivision, sharp corners and edges get smoothed out because the weighted averaging pulls vertices toward their neighbors. The more iterations you apply, the rounder everything becomes and the sharper the original features disappear.

The cube subdivides asymmetrically by default because each face of the cube only has one diagonal edge, and all the diagonals happen to run in the same rotational direction around the cube. This means the mesh is not symmetric to begin with, so the subdivision inherits and amplifies the asymmetry. To fix this, I pre-processed the cube by splitting the diagonal edge on each face so that each face has an X pattern. This makes the mesh have 4-fold symmetry per face before subdivision, and the result is a much more symmetric rounded cube after several iterations of L.
